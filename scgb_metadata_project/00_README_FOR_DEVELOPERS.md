# 单细胞基因表达元数据库项目 - 开发者指南

## 项目概述

这是一个旨在整合全球主要单细胞基因表达数据库（CellXGene、GEO、SRA、EBI、HCA等）的统一元数据平台。
已经完成数据库元数据收集，目前核心任务是数据整理模块：大型数据库元信息的二次整理开发
(1)创建统一scheme体系：根据不同数据层级建立分表（project/study-series-sample/donor-celltype）
(2)去重，建立分表的联系
(3)设计合理的可以适配后续的数据库agent开发的数据库体系

核心问题：
（1）统一数据源有不同阶段的处理形式，在不同数据库中又有不同处理版本，需要合理处理id 的映射，正确理解数据关系。
（2）不同数据库的原始数据系统整理架构和储存方式不一致，字段的详细程度也不一致，需要结合实际情况谨慎地继续设计和优化scheme中的分表和字段设计。
（3）不同数据库的数据字段标准不一样，需要进行详细的复杂的标准化工作。

### 核心挑战
1. **数据异构性**: 各数据库schema不同，字段定义不一致
2. **数据冗余**: 同一生物学样本在多个数据库中存在
3. **版本追踪**: 原始数据经多次处理产生多个版本
4. **规模巨大**: 涉及100万+样本，需高效处理

### 核心目标
构建**多表存储 + 统一视图**的元数据平台（作为后续元数据管理agent的基础服务），实现：
- 智能样本去重（基于生物学特征）
- 数据版本血缘追踪
- 跨数据库关联查询
- 可扩展的数据接入架构

### 2. 版本追踪
**决策**: 使用版本链而非简单版本号
```sql
-- 每个数据集记录包含
root_dataset_fk      -- 指向最原始数据
parent_dataset_fk    -- 指向直接父版本
version_depth        -- 距离root的深度
version_path         -- UUID数组，完整路径
```

### 3. 跨库关联
**决策**: 通过entity_links表建立关系网络
```sql
-- 关联类型
belongs_to       -- 样本属于项目
has_dataset      -- 样本拥有数据集
derived_from     -- 数据集派生关系
version_of       -- 版本关系
same_as          -- 跨库同一样本
```

## 开发建议

### 技术选型建议
- **数据库**: PostgreSQL 15+（必须支持JSONB和UUID）
- **ORM**: SQLAlchemy 2.0（类型提示友好）
- **API**: FastAPI（性能优于Flask）
- **任务队列**: Celery + Redis（大数据导入）

### 关键实现要点
1. **流式处理**: GEO等大数据源必须使用生成器
2. **事务控制**: 每1000-10000条记录提交一次
3. **错误隔离**: 单条记录失败不应影响整体
4. **幂等性**: 重复导入应自动识别
5. **进度追踪**: 支持断点续传

## 经验与教训

### 成功经验
1. **分层架构**: 存储层/处理层/服务层分离
2. **Identity Hash**: 基于生物学特征的去重更可靠
3. **版本链**: version_path数组便于快速查询血缘
4. **JSONB**: 存储变长元数据更灵活

### 遇到的坑
1. **Excel大文件**: pandas读取378K行Excel极慢，应转CSV
2. **日期格式**: 各数据库日期格式不统一
3. **多值字段**: CellXGene用"||"分隔，需专门解析
4. **内存管理**: SQLite大量写入时会锁定
