# SCeQTL-Agent V3 关键设计决策

> 本文档记录与"过度设计"的边界划分，帮助团队理解取舍

## 核心设计哲学

```
精准优先，扩展预留，拒绝过度设计

科研场景的生命线: 准确性 > 灵活性 > 智能性
```

## 我们做的 vs 我们不做的

### 1. Agent架构

| 方案 | 我们选择的 | 我们拒绝的 | 原因 |
|------|-----------|-----------|------|
| **架构模式** | Pipeline (固定流程) | Multi-Agent协作 | 可预测、可调试、可维护 |
| **规划能力** | 策略选择 (Template/Rule/LLM) | 自主规划 (ReAct/CoT) | 准确性优先，复杂推理交给LLM |
| **状态管理** | 简单上下文传递 | 复杂状态机 | 满足多轮即可，无需复杂对话 |

### 2. 知识系统

| 方案 | 我们选择的 | 我们拒绝的 | 原因 |
|------|-----------|-----------|------|
| **知识表示** | 配置化Schema知识 | 端到端神经网络 | 可解释、可维护、领域专家可参与 |
| **本体集成** | 预定义映射 (UBERON/MONDO) | 动态知识图谱构建 | 准确性可控，无需实时推理 |
| **学习机制** | 离线分析+人工审核 | 在线强化学习 | 风险可控，符合科研严谨性 |

### 3. 检索能力

| 方案 | 我们选择的 | 我们拒绝的 | 原因 |
|------|-----------|-----------|------|
| **主要检索** | SQL结构化检索 | 纯向量检索 | 精确可控，满足90%+需求 |
| **扩展检索** | 预留Vector接口 | 立即实现混合检索 | 待验证需求后再投入 |
| **结果融合** | 简单加权 (预留) | 复杂重排序模型 | 先满足基础，再优化体验 |

### 4. 推理能力

| 方案 | 我们选择的 | 我们拒绝的 | 原因 |
|------|-----------|-----------|------|
| **简单推理** | 系统内实现 (ID关联) | 全部外包给LLM | 确定性强，可验证 |
| **复杂推理** | 外包给LLM ，并设计合理的调用链| 系统内构建推理链 | 准确性不可控，维护成本高 |
| **结果解释** | 模板化 + 溯源 | 自然语言生成解释 | 可预测，避免幻觉 |

## 关键决策记录 (ADR)

### ADR-001: 不做复杂推理能力

**状态**: 已接受  
**背景**: 讨论是否构建L3知识推理能力  
**决策**: 明确排除  
**原因**:
1. 推理链越长，错误累积越多
2. 维护成本高，生物学知识更新快
3. LLM可外包推理，Agent专注提供高质量输入
4. 资源集中在检索准确性收益更高

### ADR-002: SQL为主，Vector预留

**状态**: 已接受  
**背景**: 是否立即实现混合检索  
**决策**: SQL为主，Vector预留接口  
**原因**:
1. 92%查询可被SQL满足
2. Vector检索价值待验证
3. 混合检索结果融合复杂
4. 架构上预留，验证后可快速接入

### ADR-003: Schema知识配置化

**状态**: 已接受  
**背景**: SKB应该代码化还是配置化  
**决策**: 完全配置化 (JSON/YAML)  
**原因**:
1. 领域专家可直接编辑
2. Schema演进无需发版
3. 可版本管理，可回滚
4. 多环境支持

## 技术护城河构建

### 真正的护城河不是技术复杂度，而是:

```
1. 领域知识深度
   - 单细胞生物学专业知识
   - 实验技术理解
   - 数据库Schema专业知识

2. 数据质量与覆盖
   - 12个数据源的整合
   - 75万+样本的元数据
   - 跨库关联链路

3. 准确性口碑
   - 科研场景对准确性零容忍
   - 可信赖的系统才能被采纳

4. 持续优化能力
   - 查询日志分析
   - 用户反馈闭环
   - 领域专家参与迭代
```

### 我们不追求的技术领先:

- ❌ 最复杂的Agent架构
- ❌ 最多的LLM调用
- ❌ 最炫的交互界面

### 我们追求的产品领先:

- ✅ 最准确的检索结果
- ✅ 最可信赖的数据来源
- ✅ 最符合科研流程的设计

## 实施优先级

```
P0 (必须): 精准检索 (SQL + Schema知识)
P1 (应该): 语义扩展 (本体映射 + 同义词)
P2 (可以): 混合检索 (Vector增强)
P3 (延后): 复杂推理 (知识图谱推理)
```

---

## 团队守则

1. **准确性第一**: 宁可少功能，不可不准确
2. **简单可维护**: 拒绝为了技术而技术
3. **配置优于代码**: 让领域专家能参与
4. **验证后扩展**: 新能力需数据支撑
